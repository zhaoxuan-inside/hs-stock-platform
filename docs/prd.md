# 股票数据平台 - 产品需求文档

## 1. 产品概述

hs-stock-platform 是一个前后端分离的股票数据采集与智能问答平台，集成了通义千问、DeepSeek 等AI模型，提供股票数据的实时采集、展示、分析以及智能问答功能。平台采用简约的 Notion 风格设计，支持数据缓存、定时任务管理、WebSocket实时消息推送等核心能力。

**核心目标：**
- 为用户提供便捷的股票数据查询与分析工具
- 通过集成AI模型增强数据分析能力
- 支持灵活的定时数据采集任务配置

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 账号密码登录 | 浏览股票数据、使用AI问答、管理消息 |
| 管理员 | 系统后台创建 | 所有功能权限，包括任务管理、用户管理 |

### 2.2 功能模块

1. **登录认证模块**：登录页面、权限验证、Token管理
2. **股票数据模块**：股票列表展示、K线图绘制、数据检索、关联信息展示
3. **数据采集模块**：手动触发采集、定时任务配置、任务进度展示、任务恢复
4. **AI问答模块**：内嵌AI模型页面(iframe)、多模型对比问答、内容引用、问答记录保存
5. **消息中心模块**：WebSocket消息接收、消息筛选、阅读状态管理
6. **财务数据模块**：盈利能力、营运能力、成长能力、偿债能力、现金流量、杜邦指标
7. **公司报告模块**：业绩预告、业绩快报、分红信息
8. **行业分类模块**：行业列表、成分股查询

## 4. 用户界面设计

### 4.1 设计风格

**整体风格：Notion 简约风格**

- **主色调**：#FFFFFF 背景，#1F1F1F 主文字，#6B7280 次要文字
- **强调色**：#3B82F6 蓝色（用于按钮、链接、高亮）
- **成功色**：#10B981 绿色
- **警告色**：#F59E0B 橙色
- **错误色**：#EF4444 红色

- **按钮样式**：扁平化设计，圆角8px，悬停时有轻微阴影变化
- **卡片样式**：白色背景，圆角12px，灰色边框，悬停时有阴影效果
- **字体**：系统默认无衬线字体（Inter/Segoe UI）
- **布局**：左侧侧边栏导航，右侧内容区域，响应式设计

### 4.2 SPA架构设计

**单页应用路由结构：**
`
/                           # 登录页(未认证)
/dashboard                  # 仪表盘(认证)
/stocks                     # 股票列表(认证)
/stocks/:code               # 股票详情(认证)
/tasks                      # 采集任务(认证)
/tasks/scheduler            # 定时任务(认证)
/chat                       # AI问答(认证)
/messages                   # 消息中心(认证)
/finance                    # 财务数据(认证)
/reports                    # 公司报告(认证)
/industry                   # 行业分类(认证)
`

**路由守卫策略：**
- 未认证用户只能访问 /login
- 认证用户访问 /login 自动重定向到 /dashboard
- 所有API请求携带 trace_id 和 Authorization Token

### 4.3 第三方页面嵌入设计

**AI问答页面嵌入方式：**
- 使用 iframe 内嵌通义千问、DeepSeek 等第三方页面
- 提供切换标签页在不同模型间切换
- 支持从股票页面引用内容到问答输入框
- 问答结果对比展示

**嵌入页面安全：**
- 设置 iframe sandbox 属性限制权限
- 监听 message 事件进行跨域通信
- 提供内容引用桥接功能

### 4.5 任务进度与恢复功能设计

**任务状态定义：**

| 状态 | 说明 | 操作 |
|------|------|------|
| pending | 等待执行 | 删除 |
| running | 正在执行 | 查看进度 |
| paused | 已暂停 | 继续、删除 |
| failed | 执行失败 | 重试、查看详情、删除 |
| success | 执行成功 | 查看详情、重新执行 |
| partial | 部分完成 | 继续、查看详情 |

**任务进度信息：**
- 当前进度百分比（0-100）
- 当前处理项（如：正在获取股票 sh.600001）
- 已完成数量/总数量
- 失败原因（如失败）
- 执行开始时间/预计剩余时间

**任务恢复功能：**
- 失败任务支持从断点继续执行
- 部分完成任务支持继续执行剩余部分
- 恢复时保留原任务配置和进度记录

## 5. 非功能性需求

### 5.1 性能要求
- 页面首屏加载时间 ≤ 2秒
- 股票列表响应时间 ≤ 500ms
- K线图渲染时间 ≤ 1秒（5000条数据以内）
- 接口响应时间 ≤ 1秒（常规请求）

### 5.2 可用性要求
- 系统可用性 ≥ 99.5%
- 数据采集任务失败自动重试机制（最多3次）
- 任务进度持久化，支持断点续传
- 网络异常时自动重连WebSocket

### 5.3 安全性要求
- JWT Token 过期时间 2小时
- 密码存储使用 BCrypt 加密
- 接口请求签名验证（第三方API）
- iframe sandbox 属性限制
- XSS、CSRF防护

### 5.4 可维护性要求
- 代码覆盖率 ≥ 80%
- API 文档自动生成（FastAPI）
- 日志记录完整（含trace_id）
- 前端请求统一trace_id链路追踪
- 关键代码位置日志埋点

### 5.5 日志与监控要求
- **trace_id**：所有前端请求生成唯一trace_id，贯穿整个请求链路
- **性能日志**：记录接口响应时间、数据库查询时间
- **用户行为日志**：记录用户操作路径、搜索关键词、点击行为
- **错误日志**：记录异常信息、堆栈跟踪
- **任务日志**：记录任务执行过程、进度、失败原因

## 8. 数据关系设计

### 8.1 一对一关系（融合展示）
- 股票 ↔ 行业分类
- 股票 ↔ 最新财务指标

### 8.2 一对多关系（列表展示）
- 股票 ↔ 财务数据（多条年度/季度记录）
- 股票 ↔ 业绩预告/快报（多条记录）
- 股票 ↔ 分红信息（多条记录）
- 股票 ↔ K线数据（多条记录）
- 任务 ↔ 任务执行记录（多条记录）

### 8.3 多对多关系（关联跳转）
- 行业 ↔ 股票（行业成分股）
- 指数 ↔ 股票（指数成分股）
